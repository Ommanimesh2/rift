<task type="auto">
  <name>Task 2: Create cobra root command with flags and argument validation</name>
  <files>cmd/root.go, cmd/version.go</files>
  <action>
1. Create `cmd/root.go` with root cobra command:
   - Command use: `imgdiff <image1> <image2>`
   - Short description: "Compare two container images and show file-level differences"
   - Long description: explain the tool's purpose, mention security awareness and multiple output formats
   - Args: cobra.ExactArgs(2) — require exactly two positional arguments
   - Example usage in help text:
     ```
     imgdiff nginx:1.24 nginx:1.25
     imgdiff myapp:latest myapp:v2.0
     imgdiff --format json alpine:3.18 alpine:3.19
     imgdiff --security-only ubuntu:22.04 ubuntu:24.04
     imgdiff ./old-image.tar ./new-image.tar
     ```
   - RunE function: for now, print a placeholder message showing parsed args and flags, then exit 0. Format: "Comparing [image1] vs [image2] (format: [format], platform: [platform])"

2. Define persistent flags on root command:
   - `--format` (string, default "terminal") — output format: terminal, json, markdown
   - `--security-only` (bool, default false) — show only security-relevant changes
   - `--quick` (bool, default false) — manifest-only comparison (no content download)
   - `--platform` (string, default "") — target platform for multi-arch images (e.g., linux/amd64)

3. Create `cmd/version.go`:
   - Subcommand: `imgdiff version`
   - Print version, commit hash, and build date using ldflags variables (define as package vars with defaults like "dev", "none", "unknown")
   - Use format: "imgdiff version [version] (commit: [hash], built: [date])"

4. Add `Execute()` function that runs root command and handles errors with os.Exit(1)

Do NOT validate image reference format yet — that's a concern for Phase 2 when we know the source types. Cobra's ExactArgs(2) is sufficient validation for now.
  </action>
