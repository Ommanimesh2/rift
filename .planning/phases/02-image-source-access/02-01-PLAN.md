<task type="auto">
  <name>Task 1: Add go-containerregistry and create source types with reference detection</name>
  <files>go.mod, go.sum, internal/source/source.go</files>
  <action>
  Run `go get github.com/google/go-containerregistry@latest` to add the dependency.

  Create internal/source/source.go with:
  1. Package declaration: `package source`
  2. SourceType enum: `SourceRemote`, `SourceDaemon`, `SourceTarball` (use iota)
  3. String() method on SourceType for display
  4. Options struct with Platform field (string, e.g., "linux/amd64")
  5. DetectSourceType(ref string) SourceType function:
     - If ref starts with "daemon://" → SourceDaemon
     - If file exists on disk (os.Stat succeeds) → SourceTarball
     - If ref has .tar, .tar.gz, or .tgz extension → SourceTarball
     - Otherwise → SourceRemote (default)
  6. parsePlatform(s string) (v1.Platform, error) helper:
     - Split on "/" → expect "os/arch" (e.g., "linux/amd64")
     - Return error if format invalid
     - Handle empty string (return zero Platform — means "use default")
  7. Open(ref string, opts Options) (v1.Image, error) function:
     - Detect source type
     - Delegate to openRemote/openDaemon/openTarball (defined in other files)
     - Wrap errors with source type context: fmt.Errorf("open %s image %q: %w", sourceType, ref, err)

  Do NOT create a custom Image wrapper — use v1.Image directly.
  Do NOT eagerly download layers — go-containerregistry is lazy by default.
  </action>
  <verify>go build ./... succeeds, go vet ./... clean</verify>
  <done>internal/source/source.go compiles with types, detection logic, and Open() stub that delegates to provider functions</done>
</task>

<task type="auto">
  <name>Task 2: Implement remote, daemon, and tarball providers with tests</name>
  <files>internal/source/remote.go, internal/source/daemon.go, internal/source/tarball.go, internal/source/source_test.go</files>
  <action>
  Create three provider files:

  **internal/source/remote.go** — openRemote(ref string, opts Options) (v1.Image, error):
  - Parse reference with name.ParseReference(ref)
  - Build remote.Option slice: always include remote.WithAuthFromKeychain(authn.DefaultKeychain)
  - If opts.Platform is non-empty: parse with parsePlatform(), add remote.WithPlatform()
  - Call remote.Image(parsedRef, options...) and return result
  - Import: name, v1/remote, authn

  **internal/source/daemon.go** — openDaemon(ref string, opts Options) (v1.Image, error):
  - Strip "daemon://" prefix from ref
  - Parse reference with name.NewTag(stripped) — daemon requires a tag, not a digest
  - Call daemon.Image(tag) and return result
  - Import: name, v1/daemon

  **internal/source/tarball.go** — openTarball(ref string, opts Options) (v1.Image, error):
  - Call tarball.ImageFromPath(ref, nil) — nil tag selects the first image
  - Return result
  - Import: v1/tarball

  **internal/source/source_test.go** — table-driven tests for DetectSourceType:
  - "nginx:latest" → SourceRemote
  - "docker.io/library/nginx:1.25" → SourceRemote
  - "ghcr.io/owner/repo:v1.0" → SourceRemote
  - "daemon://nginx:latest" → SourceDaemon
  - "daemon://myapp:v2" → SourceDaemon
  - "./image.tar" → SourceTarball (create temp file for os.Stat test)
  - "/tmp/backup.tar.gz" → SourceTarball (extension check, no file needed)
  - "archive.tgz" → SourceTarball (extension check)

  Use standard Go testing with t.Run subtests and testify is NOT needed — use plain if/t.Errorf.
  Do NOT test actual image pulling (that requires Docker/network) — just test detection logic.
  </action>
