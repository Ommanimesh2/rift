<feature>
  <name>Whiteout handling and multi-layer squashing</name>
  <files>internal/tree/tree.go, internal/tree/tree_test.go</files>
  <behavior>
    BuildTree takes an ordered slice of v1.Layer (bottom to top) and returns a *FileTree (wrapper around map[string]*FileNode) with the squashed result.

    FileTree struct: Entries map[string]*FileNode, with methods Size() int (entry count) and Get(path string) *FileNode.

    OCI whiteout rules:
    - ".wh.FILENAME" entry in a layer → delete "FILENAME" from the tree (file from lower layer removed)
    - ".wh..wh..opq" entry in a directory → delete ALL entries under that directory from lower layers (opaque whiteout), then add entries from current layer for that directory

    Test cases using multi-layer in-memory tars:
    - Two layers, no whiteouts: later layer overrides same-path file from earlier layer
    - Two layers, file whiteout: layer 2 has ".wh.foo.txt" → "foo.txt" from layer 1 not in result
    - Three layers, opaque whiteout: layer 2 has "dir/.wh..wh..opq" → all "dir/*" from layer 1 removed, but "dir/new.txt" from layer 2 preserved
    - Whiteout entry itself not in final tree (whiteout markers are metadata, not files)
    - Layer ordering matters: layer at index 0 is bottom, last index is top
    - File override: same path in layer 1 and layer 2 → layer 2's version wins (size, mode, content all from layer 2)
    - Nested whiteout: "a/b/.wh.c.txt" removes "a/b/c.txt" from lower layers
    - Whiteout for non-existent file: no error (silently ignored)
  </behavior>
  <implementation>
    BuildTree iterates layers in order (index 0 = bottom layer). For each layer:
    1. Call ParseLayer to get entries
    2. For each entry, check if it's a whiteout:
       - If name is ".wh..wh..opq": record the directory for opaque whiteout processing
       - If name starts with ".wh.": extract target filename, delete from tree
       - Otherwise: add/overwrite entry in tree
    3. Process opaque whiteouts: for each opaque dir, remove all existing entries with that dir prefix, then add current layer's entries for that dir

    Whiteout detection: filepath.Base(path) starts with ".wh." prefix. For opaque: filepath.Base(path) == ".wh..wh..opq". Target filename: strings.TrimPrefix(filepath.Base(path), ".wh."), reconstruct full path with filepath.Join(filepath.Dir(path), target).

    FileTree type wraps the map with accessor methods.
  </implementation>
</feature>
