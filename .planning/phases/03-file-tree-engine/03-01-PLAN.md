<feature>
  <name>Single-layer tar parsing into FileNodes</name>
  <files>internal/tree/tree.go, internal/tree/tree_test.go</files>
  <behavior>
    ParseLayer takes a v1.Layer and returns a map[string]*FileNode.

    FileNode struct contains: Path (string), Size (int64), Mode (os.FileMode), UID (int), GID (int), IsDir (bool), LinkTarget (string for symlinks), Digest (v1.Hash for content hash).

    Test cases using in-memory tar archives (archive/tar + bytes.Buffer wrapped in a fake v1.Layer):
    - Regular file with known content → FileNode with correct path, size, mode
    - Directory entry → FileNode with IsDir=true, size=0
    - Symlink → FileNode with LinkTarget set, size=0
    - File metadata preserved: mode bits (0755 vs 0644), uid/gid values
    - Multiple entries in single layer → all present in result map
    - Empty layer → empty map (no error)
    - Paths normalized: leading "./" and "/" stripped, entries keyed by clean path
  </behavior>
  <implementation>
    Package: internal/tree

    FileNode struct with fields above. Use crypto/sha256 for content digest of regular files during parsing. Use archive/tar to read layer's Uncompressed() reader.

    For testing: create a helper that builds a v1.Layer from a []tarEntry struct (path, content, mode, uid, gid, type). Use a fake v1.Layer implementation backed by bytes.Buffer — implement Uncompressed() returning the tar reader, stub other v1.Layer methods to return zero values.

    Path normalization: strings.TrimLeft(header.Name, "./") to get clean relative paths. Skip entries with empty name after normalization.

    For digest: compute sha256 of file content during parsing (read content into sha256 hasher). Store as v1.Hash{Algorithm: "sha256", Hex: hex}.
  </implementation>
</feature>
