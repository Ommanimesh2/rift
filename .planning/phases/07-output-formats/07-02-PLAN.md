  <behavior>
    FormatMarkdown(result *diff.DiffResult, image1, image2 string, events []security.SecurityEvent) string

    Output structure (exact):

    ## Image Diff: `{image1}` → `{image2}`

    ### Summary

    | Category | Count | Size |
    |----------|-------|------|
    | Added    | {N}   | {+bytes or —} |
    | Removed  | {N}   | {-bytes or —} |
    | Modified | {N}   | — |

    (byte column shows formatted size like "+12.3 KB" or "-4.5 KB"; "—" when bytes == 0)

    ### Changes

    | Status | Path | Size Delta | Details |
    |--------|------|------------|---------|
    | `+` | `{path}` | {delta} | |
    | `-` | `{path}` | {delta} | |
    | `~` | `{path}` | {delta} | {flags} |

    (Status: `+` for added, `-` for removed, `~` for modified)
    (Path wrapped in backticks)
    (Size Delta uses FormatSizeDelta, e.g. "+1.0 KB", "-512 bytes", "0 bytes")
    (Details: comma-separated flags for modified entries, empty for added/removed)

    ### Security Findings (N) — only when events is non-empty

    | Kind | Path | Mode |
    |------|------|------|
    | {kind_label} | `{path}` | |              ← for Added events (before_mode == 0)
    | {kind_label} | `{path}` | `0NNN → 0MMM` | ← for Modified events with mode info

    (kind_label uses the same securityKindLabel mapping: "SUID", "SGID", "SUID ADDED",
     "SGID ADDED", "NEW EXEC", "WORLD-WRITABLE", "PERM ESCALATION")
    (Mode in octal with 4 digits: fmt.Sprintf("0%04o → 0%04o", before, after))

    Empty diff (no entries):
    - Summary table shows zeros
    - Changes section: "(No changes)" instead of the table

    No security events (nil or empty):
    - Security Findings section is omitted entirely

    Output ends with a single trailing newline (\n).

    Exact test cases:
    1. Empty result, no events → summary zeros, "(No changes)" in changes section, no security section
    2. One added entry ("usr/bin/wget", delta +123456) → summary row "Added | 1 | +120.6 KB", changes row "| `+` | `usr/bin/wget` | +120.6 KB | |"
    3. One removed entry → summary row with negative size, changes row with "-" prefix
    4. One modified entry with ContentChanged=true, ModeChanged=true → changes row with "content, mode" in details
    5. Security event with before_mode=0 → mode column empty
    6. Security event with before_mode=0755, after_mode=04755 → mode column "`00755 → 04755`"
    7. Full report (mixed changes + security events) → all sections present, ordering maintained
  </behavior>
  <implementation>
    Create internal/output/markdown.go in package output.

    Import: fmt, strings, github.com/ommmishra/imgdiff/internal/diff, github.com/ommmishra/imgdiff/internal/security

    Use strings.Builder for efficient string construction.

    Reuse existing package-level helpers:
    - FormatBytes for size formatting
    - FormatSizeDelta for delta formatting
    - collectFlags for getting modified flags (already exported via collectFlags — actually it's unexported, so just reimplement inline or call it; since both files are in same package, collectFlags is accessible)
    - securityKindLabel for security event labels (also in same package, unexported — accessible)

    Mode display for security section: when Before != 0, format as
    fmt.Sprintf("`0%04o → 0%04o`", before, after); otherwise empty string.

    Summary size column: if AddedBytes > 0 → "+"+FormatBytes(AddedBytes), else "—".
    For removed: if RemovedBytes > 0 → "-"+FormatBytes(RemovedBytes), else "—".

    No external dependencies needed beyond standard library.
  </implementation>
