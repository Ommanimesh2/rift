Register in init():
```go

rootCmd.PersistentFlags().StringVar(&flags.password, "password", "", "registry password for explicit authentication")
rootCmd.PersistentFlags().BoolVar(&flags.exitOnChange, "exit-code", false, "exit 2 if any file changes are found")
rootCmd.PersistentFlags().BoolVar(&flags.exitOnSecurity, "fail-on-security", false, "exit 2 if security events are detected")
rootCmd.PersistentFlags().StringVar(&flags.sizeThreshold, "size-threshold", "", "exit 2 if net size increase exceeds threshold (e.g., 10MB, 500KB)")
```

Wire username/password into source.Open():
```go
opts := source.Options{
    Platform: flags.platform,
    Username: flags.username,
    Password: flags.password,
}
```

Wire exit code evaluation at the end of RunE, AFTER all output is written
(before the final `return nil`):

```go
// Parse size threshold (ignore error if empty — validated by ParseSizeThreshold).
threshold, thresholdErr := exitcode.ParseSizeThreshold(flags.sizeThreshold)
if thresholdErr != nil {
    return fmt.Errorf("invalid --size-threshold %q: %w", flags.sizeThreshold, thresholdErr)
}

ecOpts := exitcode.Options{
    ExitOnChange:   flags.exitOnChange,
    ExitOnSecurity: flags.exitOnSecurity,
    SizeThreshold:  threshold,
}
if code := exitcode.Evaluate(result, events, ecOpts); code != 0 {
    os.Exit(code)
}
```

IMPORTANT: The exit code evaluation block must go AFTER all output is written to
stdout. Place it as the last operation before `return nil` in the RunE function body.

For quick mode: exit code evaluation is not applied (quick mode is manifest-only,
no full diff result is produced). Quick mode returns early before exit code logic.

Add tests in cmd/root_test.go verifying:
- --size-threshold with invalid value returns error
- --platform flag is accepted (already exists, confirm no regression)

Avoid: Using cobra's RunE error return for the exit 2 case — that causes cobra to
print "Error: ..." to stderr and exit 1. Use os.Exit(2) directly after output is
written. This is the correct pattern for tools that distinguish "tool error" (1)
from "condition triggered" (2).
