<feature>
  <name>Security analysis engine</name>
  <files>internal/security/security.go, internal/security/security_test.go</files>
  <behavior>
    Permission bit constants (use named constants, not magic numbers):
    - ModeSetUID  = 0o4000  (SUID bit)
    - ModeSetGID  = 0o2000  (SGID bit)
    - ModeExec    = 0o111   (any execute bit)
    - ModeWorldW  = 0o002   (world-writable bit)

    SecurityEventKind (string enum):
    - "new_suid"         — Added file with SUID bit set
    - "new_sgid"         — Added file with SGID bit set
    - "suid_added"       — Modified file: SUID bit was not set in Before, is set in After
    - "sgid_added"       — Modified file: SGID bit was not set in Before, is set in After
    - "new_executable"   — Added file with any execute bit set (not a directory)
    - "world_writable"   — Added or modified file (not a directory) with world-writable bit set
    - "perm_escalation"  — Modified file: After.Mode & 0o777 > Before.Mode & 0o777 (strictly more permissive)

    SecurityEvent struct:
    - Kind     SecurityEventKind
    - Path     string
    - Before   os.FileMode (0 for Added entries — no Before node)
    - After    os.FileMode

    Analyze(result *diff.DiffResult) []SecurityEvent:
    - Iterate result.Entries
    - For each Added entry (Before == nil):
      * If After.Mode has ModeSetUID → emit "new_suid"
      * If After.Mode has ModeSetGID → emit "new_sgid"
      * If After.Mode has ModeExec AND not IsDir → emit "new_executable"
      * If After.Mode has ModeWorldW AND not IsDir → emit "world_writable"
    - For each Modified entry (Before != nil AND After != nil):
      * If Before.Mode does not have ModeSetUID AND After.Mode has ModeSetUID → emit "suid_added"
      * If Before.Mode does not have ModeSetGID AND After.Mode has ModeSetGID → emit "sgid_added"
      * If After.Mode has ModeWorldW AND not After.IsDir → emit "world_writable"
      * If After.Mode & 0o777 > Before.Mode & 0o777 → emit "perm_escalation"
    - Removed entries: no security events (removing files is not a security risk)
    - Return events in iteration order (same order as result.Entries)
    - Multiple events per entry are possible (e.g. new SUID executable → both "new_suid" and "new_executable")

    Test cases (use external test package security_test, helper funcs for building DiffEntries):
    - Added regular file with no special bits → no events
    - Added file with SUID bit (e.g. Mode=0o4755) → "new_suid" + "new_executable"
    - Added file with SGID bit (e.g. Mode=0o2755) → "new_sgid" + "new_executable"
    - Added file with only exec bit (e.g. Mode=0o755) → "new_executable" only
    - Added directory with exec bit (Mode=0o755, IsDir=true) → no events (dirs excluded)
    - Added file world-writable (Mode=0o777) → "new_executable" + "world_writable"
    - Added file world-writable non-exec (Mode=0o666) → "world_writable" only
    - Modified: SUID bit added (Before=0o755, After=0o4755) → "suid_added" + "perm_escalation"
    - Modified: SGID bit added (Before=0o755, After=0o2755) → "sgid_added" + "perm_escalation"
    - Modified: permissions loosened (Before=0o644, After=0o755) → "new_executable" is NOT emitted for Modified (only for Added) + "perm_escalation"
    - Modified: world-writable added (Before=0o644, After=0o646) → "world_writable" + "perm_escalation"
    - Modified: permissions tightened (Before=0o755, After=0o644) → no events
    - Removed entry → no events
    - Empty DiffResult → empty slice (not nil)
  </behavior>
  <implementation>
    Create internal/security/ directory and security.go.

    Use os.FileMode for permission bits — FileNode.Mode is already os.FileMode.

    The Analyze function reads diff.DiffEntry.Before and diff.DiffEntry.After (*tree.FileNode).
    For Added: Before == nil, After != nil.
    For Removed: Before != nil, After == nil.
    For Modified: both non-nil.

    Keep Analyze deterministic — no sorting beyond entry order.

    No I/O, no global state, no external dependencies beyond internal/diff and internal/tree.
  </implementation>
</feature>
