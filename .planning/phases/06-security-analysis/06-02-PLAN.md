<feature>
  <name>Security output and CLI integration</name>
  <files>internal/output/terminal.go, internal/output/terminal_test.go, cmd/root.go</files>
  <behavior>
    FormatSecurityEvent(event security.SecurityEvent) string:
    - Format: "  [KIND] path  (before: BEFORE → after: AFTER)"
    - For Added entries (Before == 0): omit the before/after modes, just "  [KIND] path"
    - Kind labels (display form):
      * "new_suid"       → "SUID"
      * "new_sgid"       → "SGID"
      * "suid_added"     → "SUID ADDED"
      * "sgid_added"     → "SGID ADDED"
      * "new_executable" → "NEW EXEC"
      * "world_writable" → "WORLD-WRITABLE"
      * "perm_escalation"→ "PERM ESCALATION"
    - Mode formatting: use octal, e.g. 0o755 → "0755"

    RenderTerminalWithSecurity(result *diff.DiffResult, layerSummary *output.LayerSummary, events []security.SecurityEvent) string:
    - Extends RenderTerminalWithLayers with an optional security section
    - If events is nil or empty: output is identical to RenderTerminalWithLayers (backward compatible)
    - If events non-empty: append after the diff entries and before/after the summary:
      ```
      Security Findings (N)
      ━━━━━━━━━━━━━━━━━━━━━
        [SUID] usr/bin/newbinary
        [WORLD-WRITABLE] etc/shadow  (before: 0640 → after: 0646)
      ```
    - Security section header styled with lipgloss bold + yellow/amber color
    - Each event line styled with red for SUID/SGID/SUID_ADDED/SGID_ADDED, yellow for others
    - Section separator: lipgloss border or simple "━" repeated line

    CLI changes in cmd/root.go:
    - After computing diff result, call security.Analyze(result)
    - Pass events to RenderTerminalWithSecurity
    - If --security-only flag is set:
      * Build a set of paths that have security events
      * Filter result.Entries to only those paths before rendering
      * Still show full security section (unfiltered)
      * If no security events at all: print "No security findings." and exit 0
    - Security analysis errors are non-fatal (like layer comparison): log to stderr, continue

    Tests (in terminal_test.go):
    - FormatSecurityEvent: Added SUID event → correct format with no mode arrows
    - FormatSecurityEvent: Modified perm_escalation → correct format with mode arrows
    - RenderTerminalWithSecurity: empty events → identical to no-security render
    - RenderTerminalWithSecurity: single event → security section appears at end
    - RenderTerminalWithSecurity: multiple events → all listed
  </behavior>

       - Filter result.Entries to only paths in events set
    4. Call output.RenderTerminalWithSecurity(result, layerSummary, events)
