<feature>
  <name>Diff engine: compare two FileTrees</name>
  <files>internal/diff/diff.go, internal/diff/diff_test.go</files>
  <behavior>
    Types:
    - ChangeType enum: Added, Removed, Modified
    - DiffEntry struct:
        Path string
        Type ChangeType
        Before *tree.FileNode (nil for Added)
        After *tree.FileNode (nil for Removed)
        SizeDelta int64 (After.Size - Before.Size; for Added = After.Size; for Removed = -Before.Size)
        Change flags (booleans, set only for Modified entries):
          ContentChanged bool (digest hex differs)
          ModeChanged bool (permission bits differ)
          UIDChanged bool
          GIDChanged bool
          LinkTargetChanged bool
          TypeChanged bool (IsDir flipped between before/after)
    - DiffResult struct:
        Entries []*DiffEntry (sorted alphabetically by Path)
        Summary counts: Added, Removed, Modified int
        Byte totals: AddedBytes, RemovedBytes int64
    - DiffResult.String() returns summary like: "5 added, 2 removed, 3 modified (+1.2 MB / -200 bytes)"

    Diff(a, b *tree.FileTree) *DiffResult algorithm:
    - For each path in b.Entries: if not in a → Added; if in both → compare attributes
    - For each path in a.Entries: if not in b → Removed
    - Comparison checks: Digest.Hex, Mode, UID, GID, IsDir, LinkTarget, Size
    - If all attributes identical → skip (no entry in result)
    - Sort result entries alphabetically by Path
    - Compute summary counters and byte totals from entries

    Test cases:
    1. Two identical single-file trees → empty result (0/0/0)
    2. Tree A empty, Tree B has 2 files → 2 Added, AddedBytes = sum of sizes
    3. Tree A has 2 files, Tree B empty → 2 Removed, RemovedBytes = sum of sizes
    4. Same path, different digest → Modified with ContentChanged=true
    5. Same path, different mode → Modified with ModeChanged=true
    6. Same path, different UID → Modified with UIDChanged=true
    7. Same path, different GID → Modified with GIDChanged=true
    8. Same path, file becomes dir → Modified with TypeChanged=true
    9. Same path, symlink target changes → Modified with LinkTargetChanged=true
    10. Mixed changes: added + removed + modified in one diff → correct counts
    11. Both trees empty → empty result
    12. Result entries sorted alphabetically by path
    13. SizeDelta correct for modified files (positive = grew, negative = shrank)
  </behavior>
  <implementation>
    Create internal/diff/ package with diff.go and diff_test.go.

    diff.go:
    - Package diff
    - Import internal/tree, fmt, sort
    - ChangeType as int with iota: Added, Removed, Modified
    - DiffEntry struct with all fields listed above
    - DiffResult struct with Entries slice, Added/Removed/Modified counts, AddedBytes/RemovedBytes
    - Diff(a, b *tree.FileTree) *DiffResult:
      1. Initialize result
      2. Range over b.Entries: lookup in a.Entries
         - Not found → append Added entry
         - Found → call compareNodes, if changed → append Modified entry
      3. Range over a.Entries: lookup in b.Entries
         - Not found → append Removed entry
      4. Sort entries by Path
      5. Compute counters from entries
    - compareNodes helper: returns *DiffEntry or nil if identical
      Compare: Digest.Hex, Mode, UID, GID, IsDir, LinkTarget
      Set individual change flags
    - DiffResult.String(): format summary with formatBytes helper (same pattern as tree.go)
    - formatBytes helper (local to diff package, same logic as tree.formatBytes)

    diff_test.go:
    - Import internal/tree (use tree.FileTree and tree.FileNode directly)
    - Helper: makeTree(entries map[string]*tree.FileNode) *tree.FileTree
    - Helper: makeNode(path, content, mode, ...) for quick FileNode construction
    - Table-driven subtests for all cases listed above
  </implementation>
