<objective>
Create `IdenticalLeadingLayers(img1, img2 v1.Image) (int, error)` and
`BuildFromImageSkipFirst(img v1.Image, skipFirst int) (*FileTree, error)` in
`internal/tree/tree.go`. These enable skipping identical leading layers
without downloading them, using only `layer.DiffID()` which reads from the
manifest JSON without any network round-trip.

Purpose: Layer downloads are the dominant cost in imgdiff. Base images like
debian or ubuntu are typically identical between versions — only the top 1-3
layers differ. By comparing DiffIDs (the uncompressed content digest from the
image config) before calling `Uncompressed()`, we can skip downloading layers
that provably contribute no diff.

Output: Two new exported functions in `internal/tree/tree.go`, tested with
table-driven TDD covering the whiteout-safety rule: only skip contiguous
identical prefix layers, never non-prefix identical layers.
</objective>

<feature>
  <name>Identical leading layer detection + skip-aware tree builder</name>
  <files>internal/tree/tree.go, internal/tree/tree_test.go</files>
  <behavior>
    IdenticalLeadingLayers(img1, img2 v1.Image) (int, error):
    - Call img1.Layers() and img2.Layers() — no download, returns lazy layer objects
    - Walk from index 0 upward; for each i call layers1[i].DiffID() and layers2[i].DiffID()
    - Stop at first mismatch or first DiffID error (treat error as "can't skip, return i, nil")
    - Return count of contiguous identical leading layers
    - Edge cases:
      * Both images empty → return 0, nil
      * One image shorter: only compare min(len1, len2) layers; when all compared are equal, return min
      * img1=[A,B,C], img2=[A,B,C,D] → return 3 (skip all three shared leading layers)
      * img1=[A,B,C], img2=[A,X,C] → return 1 (only A matches; stop at mismatch on B vs X)
      * img1=[A,B,C], img2=[A,C,B] → return 1 (only A matches; stop at B vs C mismatch)
    - Exported (called from cmd/root.go in 08-03)

    BuildFromImageSkipFirst(img v1.Image, skipFirst int) (*FileTree, error):
    - Like BuildFromImage but skips the first `skipFirst` layers
    - Call img.Layers(), then process layers[skipFirst:]
    - If skipFirst >= len(layers): return empty FileTree with Entries: make(map[string]*FileNode), nil
    - If skipFirst == 0: behaves identically to BuildFromImage

    Test cases for IdenticalLeadingLayers (use a fakeImageWithLayers helper):
    - Both empty images → 0
    - All layers identical → min(len1, len2)
    - No layers identical → 0
    - Partial prefix: first 2 of 4 identical → 2
    - Prefix only: img1=[A,B,X], img2=[A,B,Y] → 2
    - Non-contiguous shared: img1=[A,X,B], img2=[A,Y,B] → 1 (B at pos 2 should NOT be skipped)
    - Different length with full prefix overlap: img1=[A,B], img2=[A,B,C] → 2
    - DiffID error on layer i → return i, nil (non-fatal: conservative, don't skip)

    Test cases for BuildFromImageSkipFirst:
    - skipFirst=0 produces same result as BuildFromImage
    - skipFirst=1 with a 3-layer image: first layer files absent, layers 2+3 present
    - skipFirst >= len(layers): returns empty tree (no panic)
  </behavior>
