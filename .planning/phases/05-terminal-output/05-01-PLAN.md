<feature>
  <name>Diff output text formatter</name>
  <files>internal/output/format.go, internal/output/format_test.go</files>
  <behavior>
    FormatBytes(n int64) string:
    - 0 → "0 bytes"
    - 500 → "500 bytes"
    - 1024 → "1.0 KB"
    - 1536 → "1.5 KB"
    - 1048576 → "1.0 MB"
    - 1073741824 → "1.0 GB"

    FormatSizeDelta(delta int64) string:
    - 1024 → "+1.0 KB"
    - -2048 → "-2.0 KB"
    - 0 → "0 bytes"

    FormatEntry(entry *diff.DiffEntry) string:
    - Added entry (path="usr/bin/foo", SizeDelta=1024):
      → "+ usr/bin/foo  (+1.0 KB)"
    - Removed entry (path="usr/lib/old.so", SizeDelta=-2048):
      → "- usr/lib/old.so  (-2.0 KB)"
    - Modified entry (path="etc/config", ContentChanged=true, ModeChanged=true, SizeDelta=0):
      → "~ etc/config  [content, mode]  (0 bytes)"
    - Modified entry with only content change and positive delta:
      → "~ etc/app.conf  [content]  (+512 bytes)"
    - Change flag labels: content, mode, uid, gid, link, type (matching DiffEntry bool fields)

    FormatSummary(result *diff.DiffResult) string:
    - 5 added, 2 removed, 3 modified with byte totals:
      → "5 added (+12.3 MB), 2 removed (-256 KB), 3 modified"
    - All zeros → "0 added, 0 removed, 0 modified"
    - No byte deltas → omit parenthetical for zero sections

    Render(result *diff.DiffResult) string:
    - Combines all FormatEntry lines (one per entry, newline separated)
    - Blank line
    - FormatSummary line
    - Empty DiffResult → just summary "0 added, 0 removed, 0 modified"

    Tests should use the same makeTree/makeNode helper pattern from diff_test.go for constructing test DiffResults. Use external test package (output_test).
  </behavior>
  <implementation>
    Create internal/output/format.go with package output. Import diff and tree packages.

    FormatBytes: export the same logic currently duplicated in tree.go and diff.go. Use binary SI (1024-based) with KB/MB/GB suffixes.

    FormatSizeDelta: prepend "+" for positive, "-" for negative (already has sign), no prefix for zero.

    FormatEntry: use "+" prefix for Added, "-" for Removed, "~" for Modified. Right-align or pad the size delta. For Modified entries, collect active change flags into a bracketed comma-separated list: [content, mode, uid, gid, link, type].

    FormatSummary: format each count with its byte total in parentheses. Omit byte parenthetical if the byte count is 0 for that category. Modified entries contribute to neither added nor removed bytes — show just the count.

    Render: iterate Entries, FormatEntry each, join with newlines, append blank line + FormatSummary.
  </implementation>
</feature>
